The DeclarationName class represents the name of a declaration. 
- Separate class as variable declaration can take many forms.
- E.g. `f(int x)` has variable declarations `f` and `x`.
- These also include class names, class constructor, destructor, and operation overloaded names, and more.
- For a `DeclarationName` `N`, the getter function `N.getNameKind()` will return what kind of name `N` stores. there are 10 different possibilities:
	- `Identifier` : `N` is a simple variable. `N.getAsIdentifierInfo()` will retrieve the corresponding `IdentifierInfo*` pointer that points towards the actual identifier.
	- `ObjCZeroArgSelector`, `ObjCOneArgSelector`, `ObjCMultiArgSelector`: **Only for Objective C**. 
	- `CXXConstructorName`: `N` is a C++ Constructor Name.`N.getCXXNameType()` will return the type that the constructor will construct, which will always be canonical.
	- `CXXDestructorName`: `N` is a C++ Destructor Name. `N.getCXXNameType()` does the same thing as with constructors.
	- `CXXConversionFunctionName`: `N` is a C++ Conversion name (cast overload: `operator type const *`). `N.getCXXNameType()` will return the target type.
	- `CXXOperatorName`: `N` is a overloaded operator. `N.getCXXOverloadedOperator()` will return the overloaded operator.
	- `CXXLiteralOperatorName`: `N` is a user-defined literal operator (C++11 and newer). these are implemented with the syntax `operator"" _suffix`, and `N.getCXXLiteralIdentifier()` is used to get the corresponding `IdentifierInfo*` pointing to the identifier.
	- `CXXUsingDirective`: `N` is a C++ using directive. 
- `DeclarationName`s are cheap to copy, create, and compare (usually only a single pointer)
- Ordering operators are overloaded
